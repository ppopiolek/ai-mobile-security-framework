Overview

This site summarizes the article “Threat Modeling and a Systematic Security Framework for AI‑Native Next‑Generation Mobile Networks”. It provides a concise view on network‑oriented Model Cards, an AI‑specific threat modeling approach, and a five‑stage security framework for AI components embedded in 5G/6G systems.

Model Cards (Network‑Oriented)

Model Cards document an AI/ML model together with its network context: placement (e.g., NWDAF, MDAF, RIC), data sources/clients (NFs), inputs/outputs, validation, and environment.

How to add/edit Model Cards

Update models.json in this repository. The site will render cards automatically.

Threat Modeling (AI‑Specific)

The approach characterizes the attacker by knowledge (model, data, environment), capabilities, and goals, and standardizes TTPs using matrices (e.g., MITRE FiGHT / ATT&CK, OWASP ML). Risk is prioritized via three views:

  • Probability Matrix — attacker’s cost vs. benefit.
  • Risk Matrix — likelihood vs. consequences (defender view).
  • Priority Matrix — risk vs. defense cost.

This yields practical and context‑aware scenarios aligned with mobile network realities.

Security Framework (Five Stages)

  1. Input — accept model + Model Card with integration details (NFs, data flow).
  2. Key Threats Identification — system context, threat intel, attacker modeling, risk analysis.
  3. Evaluation Environment Setup — configure representative scenarios (e.g., in an NDT/testbed).
  4. Security Testing — assess robustness under standard and adversarial conditions.
  5. Continuous Monitoring — production policies, drift checks, feedback loop to testing.

Case Study (Example)

Example: LSTM Autoencoder anomaly detection in NWDAF using realistic traffic data. Coordinated traffic manipulation can degrade detection integrity with limited signals in anomaly scores, underscoring the need for NDT‑based testing, XAI‑aided validation, and continuous monitoring.
